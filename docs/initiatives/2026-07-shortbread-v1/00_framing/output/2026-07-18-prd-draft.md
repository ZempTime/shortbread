# Shortbread v1 PRD — Framing Draft

## Problem Statement

Sharing interactive HTML outside a producer's workspace currently forces a poor choice between public hosting, screenshots, account ceremony, or one-off infrastructure. The Owner needs to share personal multi-page Sites, iterate on them, make them deliberately available offline, and recover contextual Viewer feedback without turning Shortbread into a CMS, collaboration suite, notification service, or AI processor.

## Solution

Build a public MIT-licensed, single-Owner Rails application and Go CLI. Producers publish already-built Bundles as immutable, content-addressed Releases. Named Viewers enter through preview-safe personal Invitations, see their Sites on an apex Shelf, open each Site on an isolated stable origin, optionally keep a complete Release offline, and write to one flat Release/path-anchored Feedback Thread. Operators can provision the reference Northflank + PlanetScale Postgres + private R2 deployment from credentials and values through tested, idempotent automation.

## Story Groups

The canonical PRD expands these into numbered acceptance stories:

1. Owner bootstrap, passkeys, recovery, Sites, People, Grants, Invitations, Releases, rollback, offline policy, feedback, receipts, tokens, and deletion.
2. Viewer one-tap acceptance, Shelf, host isolation, optional passkey re-entry, deliberate atomic Offline Copies, Comments, downloads, and honest revocation.
3. Producer remote login/profile support, scoped CI tokens, safe directory validation, content hashing, missing-Blob uploads, atomic finalize, stable JSON, feedback retrieval, and API compatibility.
4. Operator clean clone, dependency audit/freeze, credential wizard, idempotent plan/apply/resume, secrets, deployment roles, health/migrations, backup/restore, upgrade/rollback, release artifacts, and live smoke.
5. Contributor public documentation, synthetic example Bundle, repeatable screenshots, behavioral test seams, independent review/repair, and MWP/factory harvest evidence.

## Implementation Decisions

- Rails 8.1, Inertia/React/TypeScript, PostgreSQL, private S3-compatible Blobs, self-hosted AnyCable, and Go CLI; Northflank, PlanetScale Postgres, and R2 are the reference deployment.
- One Owner forever; People and per-Site Grants; invite-only v1; no password, email, SMS, push, or anonymous public access.
- Apex control/Shelf origin and stable `<slug>.sites.<apex>` Site origins; preview-safe Invitation POST consumption; signed short-lived site handoff; host-only cookies; passkey re-entry.
- Immutable Releases, path-to-hash Manifests, direct presigned missing-Blob PUT, atomic finalize/current pointer, safe path/reserved-file/secret-like validation, exact/directory/optional-SPA routing, ranges, ETags, and private serving.
- Viewer-initiated atomic Offline Copies with required/optional entries, visible size/progress/updates/removal, browser-eviction honesty, and explicit survival after revocation.
- One injected flat append-only Feedback Thread and Owner-only View Receipts, both anchored to Release/path where relevant.
- Front-loaded open-source dependency baseline with committed lockfiles and controller-owned exceptions only.
- Public clean-clone/setup/operations/security/contributing docs, released app/container and CLI artifacts, real screenshot harness, and synthetic example Bundle.

## Testing Decisions

- Browser system tests at Owner/Viewer seams, including apex↔Site auth, feedback, install/update/offline, and screenshot capture.
- Rails request tests for WebAuthn, Invitations, host routing, API auth, publish finalize, ranges, and private serving.
- Black-box Go CLI tests against the Rails test application for remote login, publish, invite, and feedback.
- Pure Ruby/Go units only for deterministic path, Manifest, delta, cache-selection, and Blob-GC algorithms.
- Provisioner command-contract tests and clean-clone rehearsal without credentials; optional live provider smoke after credentials are supplied.
- Independent Standards and Spec review, plus security/operations review for sensitive slices, followed by repair and rereview before integration.

## Out of Scope

CMS/build steps; multiple Owners or shared SaaS tenancy; anonymous public Sites; notification delivery; feedback replies/reactions/states/assignments; group-visible receipts or third-party analytics; Shortbread-controlled AI; public-link grants; native apps; historical-release hostnames; source `pull`/round-trip export; custom roles/tiers; password/email/SMS auth; automatic removal of saved Offline Copies; custom deployment providers beyond documented standard seams.
