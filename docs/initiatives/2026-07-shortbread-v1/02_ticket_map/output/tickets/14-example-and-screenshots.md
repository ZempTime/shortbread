# Ship a public-source tour and repeatable screenshots

Parent: #1
Blocked by: #8, #9, #10, #11
Local ticket ID: T14

## Outcome

A deterministic invented-data Bundle and real-browser tour demonstrate publish, Invitation, Shelf, private Site, offline keeping/update, feedback, View Receipts, republish, and deletion; a repeatable harness captures current documentation screenshots.

## Acceptance

- Example source is small, attractive, multi-page, offline-capable, openly licensed, contains no real private/proprietary data, and remains invite-only when hosted.
- One scripted fixture seed/publish/tour reaches every named state deterministically and is usable in local docs and clean-room rehearsal.
- Browser capture uses the real app at named desktop/mobile viewports, deterministic clock/data, stable filenames, and accessible captions/alt text.
- Screenshot manifest maps image → flow/state/viewport; freshness check fails when required images are absent/stale.
- Tour/docs explicitly distinguish public source/evidence from anonymous public Site access.

## Evidence and review

- Example validation/publish/offline test, tour system test, screenshot regeneration/freshness checks, visual/accessibility review.
- Standards + Spec + privacy/license review, committed generated images/source and harvest.
