# Domain Docs

Shortbread is a single-context repository.

Before shaping or implementing work, read:

- `CONTEXT.md` for canonical product vocabulary;
- relevant accepted decisions under `docs/adr/`;
- the active initiative's `RUN.md` before opening any stage contract.

For staged execution and authority ownership, follow [`mwp.md`](mwp.md).

Use glossary terms in specs, issues, tests, UI copy, and code. If a needed concept is absent, resolve the language through domain modeling instead of silently inventing a synonym.

If proposed work contradicts an ADR, surface the conflict and reopen the decision explicitly. Do not silently override accepted architecture.

`CONTEXT.md` is a glossary, not an implementation spec. Implementation and workflow decisions belong in ADRs, issue bodies, or initiative stage outputs.
